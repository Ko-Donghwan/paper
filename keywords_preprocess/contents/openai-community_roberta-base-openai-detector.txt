<span>
		Model Details
	</span>
</h2>
<p><strong>Model Description:</strong> RoBERTa base OpenAI Detector is the GPT-2 output detector model, obtained by fine-tuning a RoBERTa base model with the outputs of the 1.5B-parameter GPT-2 model. The model can be used to predict if text was generated by a GPT-2 model. This model was released by OpenAI at the same time as OpenAI released the weights of the <a href="https://huggingface.co/gpt2-xl" rel="nofollow">largest GPT-2 model</a>, the 1.5B parameter version. </p>
<ul>
<li><strong>Developed by:</strong> OpenAI, see <a href="https://github.com/openai/gpt-2-output-dataset/tree/master/detector" rel="nofollow">GitHub Repo</a> and <a href="https://d4mucfpksywv.cloudfront.net/papers/GPT_2_Report.pdf" rel="nofollow">associated paper</a> for full author list</li>
<li><strong>Model Type:</strong> Fine-tuned transformer-based language model</li>
<li><strong>Language(s):</strong> English</li>
<li><strong>License:</strong> MIT</li>
<li><strong>Related Models:</strong> <a href="https://huggingface.co/roberta-base" rel="nofollow">RoBERTa base</a>, <a href="https://huggingface.co/gpt2-xl" rel="nofollow">GPT-XL (1.5B parameter version)</a>, <a href="https://huggingface.co/gpt2-large" rel="nofollow">GPT-Large (the 774M parameter version)</a>, <a href="https://huggingface.co/gpt2-medium" rel="nofollow">GPT-Medium (the 355M parameter version)</a> and <a href="https://huggingface.co/gpt2" rel="nofollow">GPT-2 (the 124M parameter version)</a></li>

<span>
		Direct Use
	</span>
</h4>
<p>The model is a classifier that can be used to detect text generated by GPT-2 models. However, it is strongly suggested not to use it as a ChatGPT detector for the purposes of making grave allegations of academic misconduct against undergraduates and others, as this model might give inaccurate results in the case of ChatGPT-generated input.</p>

<span>
		Misuse and Out-of-scope Use
	</span>
</h4>
<p>The model should not be used to intentionally create hostile or alienating environments for people. In addition, the model developers discuss the risk of adversaries using the model to better evade detection in their <a href="https://d4mucfpksywv.cloudfront.net/papers/GPT_2_Report.pdf" rel="nofollow">associated paper</a>, suggesting that using the model for evading detection or for supporting efforts to evade detection would be a misuse of the model. </p>

<span>
		Risks and Limitations
	</span>
</h4>
<p>In their <a href="https://d4mucfpksywv.cloudfront.net/papers/GPT_2_Report.pdf" rel="nofollow">associated paper</a>, the model developers discuss the risk that the model may be used by bad actors to develop capabilities for evading detection, though one purpose of releasing the model is to help improve detection research. </p>
<p>In a related <a href="https://openai.com/blog/gpt-2-1-5b-release/" rel="nofollow">blog post</a>, the model developers also discuss the limitations of automated methods for detecting synthetic text and the need to pair automated detection tools with other, non-automated approaches. They write: </p>
<blockquote>
<p>We conducted in-house detection research and developed a detection model that has detection rates of ~95% for detecting 1.5B GPT-2-generated text. We believe this is not high enough accuracy for standalone detection and needs to be paired with metadata-based approaches, human judgment, and public education to be more effective. </p>
</blockquote>
<p>The model developers also <a href="https://openai.com/blog/gpt-2-1-5b-release/" rel="nofollow">report</a> finding that classifying content from larger models is more difficult, suggesting that detection with automated tools like this model will be increasingly difficult as model sizes increase. The authors find that training detector models on the outputs of larger models can improve accuracy and robustness. </p>

<span>
		Bias
	</span>
</h4>
<p>Significant research has explored bias and fairness issues with language models (see, e.g., <a href="https://aclanthology.org/2021.acl-long.330.pdf" rel="nofollow">Sheng et al. (2021)</a> and <a href="https://dl.acm.org/doi/pdf/10.1145/3442188.3445922" rel="nofollow">Bender et al. (2021)</a>). Predictions generated by RoBERTa base and GPT-2 1.5B (which this model is built/fine-tuned on) can include disturbing and harmful stereotypes across protected classes; identity characteristics; and sensitive, social, and occupational groups (see the <a href="https://huggingface.co/roberta-base" rel="nofollow">RoBERTa base</a> and <a href="https://huggingface.co/gpt2-xl" rel="nofollow">GPT-2 XL</a> model cards for more information). The developers of this model discuss these issues further in their <a href="https://d4mucfpksywv.cloudfront.net/papers/GPT_2_Report.pdf" rel="nofollow">paper</a>.</p>

<p>The model is a sequence classifier based on RoBERTa base (see the <a href="https://huggingface.co/roberta-base" rel="nofollow">RoBERTa base model card</a> for more details on the RoBERTa base training data) and then fine-tuned using the outputs of the 1.5B GPT-2 model (available <a href="https://github.com/openai/gpt-2-output-dataset" rel="nofollow">here</a>).</p>

<span>
		Training Procedure
	</span>
</h4>
<p>The model developers write that: </p>
<blockquote>
<p>We based a sequence classifier on RoBERTaBASE (125 million parameters) and fine-tuned it to classify the outputs from the 1.5B GPT-2 model versus WebText, the dataset we used to train the GPT-2 model.</p>
</blockquote>
<p>They later state: </p>
<blockquote>
<p>To develop a robust detector model that can accurately classify generated texts regardless of the sampling method, we performed an analysis of the model’s transfer performance.</p>
</blockquote>
<p>See the <a href="https://d4mucfpksywv.cloudfront.net/papers/GPT_2_Report.pdf" rel="nofollow">associated paper</a> for further details on the training procedure.</p>

<span>
		Results
	</span>
</h4>
<p>The model developers <a href="https://d4mucfpksywv.cloudfront.net/papers/GPT_2_Report.pdf" rel="nofollow">find</a>: </p>
<blockquote>
<p>Our classifier is able to detect 1.5 billion parameter GPT-2-generated text with approximately 95% accuracy...The model’s accuracy depends on sampling methods used when generating outputs, like temperature, Top-K, and nucleus sampling (<a href="https://arxiv.org/abs/1904.09751" rel="nofollow">Holtzman et al., 2019</a>. Nucleus sampling outputs proved most difficult to correctly classify, but a detector trained using nucleus sampling transfers well across other sampling methods. As seen in Figure 1 [in the paper], we found consistently high accuracy when trained on nucleus sampling. 	</p>
</blockquote>
<p>See the <a href="https://d4mucfpksywv.cloudfront.net/papers/GPT_2_Report.pdf" rel="nofollow">associated paper</a>, Figure 1 (on page 14) and Figure 2 (on page 16) for full results.</p>
