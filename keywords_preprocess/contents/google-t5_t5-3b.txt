<span>
		Model Description
	</span>
</h2>
<p>The developers of the Text-To-Text Transfer Transformer (T5) <a href="https://ai.googleblog.com/2020/02/exploring-transfer-learning-with-t5.html" rel="nofollow">write</a>: </p>
<blockquote>
<p>With T5, we propose reframing all NLP tasks into a unified text-to-text-format where the input and output are always text strings, in contrast to BERT-style models that can only output either a class label or a span of the input. Our text-to-text framework allows us to use the same model, loss function, and hyperparameters on any NLP task.</p>
</blockquote>
<p>T5-3B is the checkpoint with 3 billion parameters. </p>
<ul>
<li><strong>Developed by:</strong> Colin Raffel, Noam Shazeer, Adam Roberts, Katherine Lee, Sharan Narang, Michael Matena, Yanqi Zhou, Wei Li, Peter J. Liu. See <a href="https://jmlr.org/papers/volume21/20-074/20-074.pdf" rel="nofollow">associated paper</a> and <a href="https://github.com/google-research/text-to-text-transfer-transformer#released-model-checkpoints" rel="nofollow">GitHub repo</a></li>
<li><strong>Model type:</strong> Language model</li>
<li><strong>Language(s) (NLP):</strong> English, French, Romanian, German</li>
<li><strong>License:</strong> Apache 2.0</li>
<li><strong>Related Models:</strong> <a href="https://huggingface.co/models?search=t5" rel="nofollow">All T5 Checkpoints</a></li>
<li><strong>Resources for more information:</strong><ul>
<li><a href="https://jmlr.org/papers/volume21/20-074/20-074.pdf" rel="nofollow">Research paper</a></li>
<li><a href="https://ai.googleblog.com/2020/02/exploring-transfer-learning-with-t5.html" rel="nofollow">Google's T5 Blog Post</a> </li>
<li><a href="https://github.com/google-research/text-to-text-transfer-transformer" rel="nofollow">GitHub Repo</a></li>
<li><a href="https://huggingface.co/docs/transformers/model_doc/t5" rel="nofollow">Hugging Face T5 Docs</a></li>
</ul>
</li>
</ul>

<span>
		Direct Use and Downstream Use
	</span>
</h2>
<p>The developers write in a <a href="https://ai.googleblog.com/2020/02/exploring-transfer-learning-with-t5.html" rel="nofollow">blog post</a> that the model: </p>
<blockquote>
<p>Our text-to-text framework allows us to use the same model, loss function, and hyperparameters on any NLP task, including machine translation, document summarization, question answering, and classification tasks (e.g., sentiment analysis). We can even apply T5 to regression tasks by training it to predict the string representation of a number instead of the number itself.</p>
</blockquote>
<p>See the <a href="https://ai.googleblog.com/2020/02/exploring-transfer-learning-with-t5.html" rel="nofollow">blog post</a> and <a href="https://jmlr.org/papers/volume21/20-074/20-074.pdf" rel="nofollow">research paper</a> for further details.</p>

<span>
		Training Data
	</span>
</h2>
<p>The model is pre-trained on the <a href="https://www.tensorflow.org/datasets/catalog/c4" rel="nofollow">Colossal Clean Crawled Corpus (C4)</a>, which was developed and released in the context of the same <a href="https://jmlr.org/papers/volume21/20-074/20-074.pdf" rel="nofollow">research paper</a> as T5.</p>
<p>The model was pre-trained on a on a <strong>multi-task mixture of unsupervised (1.) and supervised tasks (2.)</strong>.
Thereby, the following datasets were being used for (1.) and (2.):</p>
<ol>
<li><strong>Datasets used for Unsupervised denoising objective</strong>:</li>
</ol>
<ul>
<li><a href="https://huggingface.co/datasets/c4" rel="nofollow">C4</a></li>
<li><a href="https://huggingface.co/datasets/wiki_dpr" rel="nofollow">Wiki-DPR</a></li>
</ul>
<ol start="2">
<li><strong>Datasets used for Supervised text-to-text language modeling objective</strong></li>
</ol>
<ul>
<li>Sentence acceptability judgment<ul>
<li>CoLA <a href="https://arxiv.org/abs/1805.12471" rel="nofollow">Warstadt et al., 2018</a></li>
</ul>
</li>
<li>Sentiment analysis <ul>
<li>SST-2 <a href="https://nlp.stanford.edu/~socherr/EMNLP2013_RNTN.pdf" rel="nofollow">Socher et al., 2013</a></li>
</ul>
</li>
<li>Paraphrasing/sentence similarity<ul>
<li>MRPC <a href="https://aclanthology.org/I05-5002" rel="nofollow">Dolan and Brockett, 2005</a></li>
<li>STS-B <a href="https://arxiv.org/abs/1708.00055" rel="nofollow">Ceret al., 2017</a></li>
<li>QQP <a href="https://quoradata.quora.com/First-Quora-Dataset-Release-Question-Pairs" rel="nofollow">Iyer et al., 2017</a></li>
</ul>
</li>
<li>Natural language inference<ul>
<li>MNLI <a href="https://arxiv.org/abs/1704.05426" rel="nofollow">Williams et al., 2017</a></li>
<li>QNLI <a href="https://arxiv.org/abs/1606.05250" rel="nofollow">Rajpurkar et al.,2016</a></li>
<li>RTE <a href="https://link.springer.com/chapter/10.1007/11736790_9" rel="nofollow">Dagan et al., 2005</a> </li>
<li>CB <a href="https://semanticsarchive.net/Archive/Tg3ZGI2M/Marneffe.pdf" rel="nofollow">De Marneff et al., 2019</a></li>
</ul>
</li>
<li>Sentence completion<ul>
<li>COPA <a href="https://www.researchgate.net/publication/221251392_Choice_of_Plausible_Alternatives_An_Evaluation_of_Commonsense_Causal_Reasoning" rel="nofollow">Roemmele et al., 2011</a></li>
</ul>
</li>
<li>Word sense disambiguation<ul>
<li>WIC <a href="https://arxiv.org/abs/1808.09121" rel="nofollow">Pilehvar and Camacho-Collados, 2018</a></li>
</ul>
</li>
<li>Question answering<ul>

<span>
		Training Procedure
	</span>
</h2>
<p>In their <a href="https://jmlr.org/papers/volume21/20-074/20-074.pdf" rel="nofollow">abstract</a>, the model developers write: </p>
<blockquote>
<p>In this paper, we explore the landscape of transfer learning techniques for NLP by introducing a unified framework that converts every language problem into a text-to-text format. Our systematic study compares pre-training objectives, architectures, unlabeled datasets, transfer approaches, and other factors on dozens of language understanding tasks. </p>
</blockquote>
<p>The framework introduced, the T5 framework, involves a training procedure that brings together the approaches studied in the paper. See the <a href="https://jmlr.org/papers/volume21/20-074/20-074.pdf" rel="nofollow">research paper</a> for further details.</p>
